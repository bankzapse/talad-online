# ขั้นตอนที่ต้องทำเอง ก่อนเปิดใช้จริง

งานที่เหลือทั้งหมดต้องทำในบัญชีของคุณ (LINE / Supabase / ราชการ) — ไล่ตามลำดับนี้ได้เลย
เรียงจาก **ต้องทำก่อนเปิดให้คนนอกใช้** → **ทำได้ทีหลัง**

> หมายเหตุ: หน้าจอของ LINE และ Supabase มีการปรับ UI อยู่เรื่อย ๆ
> ชื่อเมนูอาจไม่ตรงเป๊ะ แต่ลำดับขั้นตอนและสิ่งที่ต้องหาจะเหมือนเดิม

---

## 🔴 1. Publish LINE Login channel — ไม่ทำ คนนอกล็อกอินไม่ได้เลย

**อาการตอนนี้:** channel อยู่สถานะ `Developing` → มีแต่คุณกับ tester ที่ล็อกอินได้
คนอื่นกดเข้าสู่ระบบจะเจอ error

**ขั้นตอน**

1. เข้า https://developers.line.biz/console/
2. เลือก Provider ของคุณ → เลือก channel **ตลาดออนไลน์** (Channel ID `2010758544`)
3. อยู่แท็บ **Basic settings**
4. มองหาสถานะ **`Developing`** ด้านบน (ป้ายสีเทา/ส้ม ข้างชื่อ channel)
5. กดปุ่ม **Publish** ข้าง ๆ สถานะนั้น
6. ยืนยันใน dialog → สถานะเปลี่ยนเป็น **`Published`**

**ก่อน publish ต้องกรอกให้ครบ** (LINE บังคับ ไม่งั้นปุ่มจะกดไม่ได้)

| ช่อง | ใส่อะไร |
|---|---|
| Channel icon | โลโก้ร้าน (สี่เหลี่ยมจัตุรัส ≥ 500×500 px) |
| Channel description | เช่น "ตลาดออนไลน์ — ซื้อขายของสดและสินค้ามือสองจากตลาดใกล้บ้าน" |
| Email address | อีเมลติดต่อของบริษัท |
| Privacy policy URL | `https://talad-online.vercel.app/privacy` |
| Terms of use URL | `https://talad-online.vercel.app/terms` |

**เช็คว่าสำเร็จ:** เปิด https://talad-online.vercel.app/login ในหน้าต่าง incognito
แล้วกดเข้าสู่ระบบด้วย LINE — ต้องเข้าหน้า LINE ได้โดยไม่ขึ้น error

## 🔴 2. LINE Messaging API token — ไม่ทำ ระบบแจ้งเตือนจะเงียบทั้งหมด

**อาการตอนนี้:** โค้ดแจ้งเตือนพร้อมแล้ว (ออร์เดอร์ใหม่ / ร้านยืนยัน / เลขพัสดุ / อนุมัติประกาศ)
แต่ไม่มี token → ระบบเขียนลง log แทนการส่งจริง ไม่มีใครได้รับข้อความ

### ⚠️ ข้อสำคัญที่พลาดกันบ่อย — ต้องอยู่ Provider เดียวกัน

LINE ให้ `userId` แบบ **ผูกกับ Provider** ไม่ใช่ผูกกับ channel

- ถ้า **Login channel** กับ **Messaging API channel** อยู่ Provider **เดียวกัน** → userId ตรงกัน ส่งข้อความได้
- ถ้าอยู่ **คนละ Provider** → userId คนละชุด **ระบบจะส่งข้อความไม่ถึงใครเลย** ทั้งที่ไม่ error

**ก่อนสร้าง ให้เช็คก่อนว่า Messaging API channel อยู่ Provider เดียวกับ channel `2010758544`**
ถ้าคนละอัน ต้องสร้าง Messaging API channel ใหม่ใน Provider เดียวกัน

### ขั้นตอน

1. เข้า https://developers.line.biz/console/
2. เลือก **Provider เดียวกับ Login channel**
3. ถ้ายังไม่มี Messaging API channel → **Create a new channel** → เลือก **Messaging API**
   (ถ้ามีอยู่แล้วจากตอนสร้าง LINE OA ให้ข้ามไปข้อ 4)
4. เข้า channel นั้น → แท็บ **Messaging API**
5. เลื่อนหาหัวข้อ **Channel access token (long-lived)**
6. กด **Issue** → คัดลอกค่าที่ได้ (ยาวมาก ~170 ตัวอักษร)

### เอา token มาใส่ระบบ

```bash
npx vercel env add LINE_CHANNEL_ACCESS_TOKEN production
# วาง token ที่คัดลอกมา แล้ว Enter

npx vercel --prod
```

หรือทำผ่านหน้าเว็บ: Vercel → project **talad-online** → Settings → Environment Variables
→ Add New → Key `LINE_CHANNEL_ACCESS_TOKEN` → Environment เลือก **Production** → Save → Redeploy

### ตั้งค่าเพิ่มที่ LINE OA (ไม่งั้นบอทตอบอัตโนมัติกวนผู้ใช้)

เข้า https://manager.line.biz/ → เลือก OA → ตั้งค่า → การตอบกลับ

- ปิด **ข้อความตอบกลับอัตโนมัติ** (auto-reply)
- ปิด **ข้อความทักทายเพื่อนใหม่** (greeting) หรือแก้ข้อความให้เข้ากับบริการ
- เปิด **Webhook** ถ้าจะทำบอทโต้ตอบทีหลัง (ตอนนี้ยังไม่ต้อง)

### ⚠️ ผู้รับต้องเป็นเพื่อนกับ OA ก่อน

LINE ห้ามส่ง push หาคนที่ไม่ได้เพิ่ม OA เป็นเพื่อน
→ ต้องให้ผู้ขาย/ผู้ซื้อกด **เพิ่มเพื่อน** LINE OA ก่อน ไม่งั้นข้อความไม่ถึง

แนะนำ: เอา QR code ของ OA ไปแปะที่หน้า `/help` และหน้าสมัครผู้ขาย

**เช็คว่าสำเร็จ:** ลองสั่งซื้อสินค้า 1 รายการ → ผู้ขายต้องได้ LINE แจ้ง "🛒 มีรายการสั่งซื้อใหม่!"

## 🟠 3. หมุน Supabase service_role key — คีย์นี้เคยถูกวางในแชท

**ทำไมต้องทำ:** `service_role` ข้าม RLS ได้ทั้งหมด = อ่าน/แก้/ลบข้อมูลทุกตารางได้
รวมถึงที่อยู่และเบอร์ลูกค้าทุกคน ใครได้คีย์นี้ไปคือได้ฐานข้อมูลทั้งใบ

**ขั้นตอน**

1. เข้า https://supabase.com/dashboard → เลือกโปรเจกต์
2. **Project Settings** (เฟืองล่างซ้าย) → **API Keys**
3. หาแถว **`service_role`** → กด **Reveal** เพื่อดูคีย์ปัจจุบัน
4. มองหาปุ่ม **Rotate / Regenerate** (บางบัญชีอยู่ใต้เมนู `...` ท้ายแถว)
5. กด rotate → คัดลอกคีย์ใหม่ทันที

**คีย์เก่าจะใช้ไม่ได้ทันทีที่ rotate → เว็บจะล่มจนกว่าจะอัปเดต** ทำต่อทันที:

```bash
npx vercel env rm SUPABASE_SERVICE_ROLE_KEY production
npx vercel env add SUPABASE_SERVICE_ROLE_KEY production
# วางคีย์ใหม่

npx vercel --prod
```

แล้วแก้ไฟล์ `.env.local` ในเครื่องให้เป็นคีย์ใหม่ด้วย

**เช็คว่าสำเร็จ:**
```bash
curl -s https://talad-online.vercel.app/api/health
# ต้องได้ "database":"ok"
```

> ⚠️ ถ้ารอตรงนี้ไว้ทีหลังก็ได้ แต่ **ต้องทำก่อนมีข้อมูลลูกค้าจริงในระบบ**

## 🟠 4. SMS OTP — ยืนยันเบอร์ผู้ขาย

**ทำไมต้องมี:** ตอนนี้ "1 เบอร์ = 1 สิทธิ์ทดลองฟรี" กันไม่ได้จริง
คนเดียวสมัครหลายบัญชีเพื่อรับทดลองฟรีซ้ำได้เรื่อย ๆ

**ผู้ให้บริการในไทยที่ใช้กันบ่อย** (ตรวจราคาปัจจุบันเองอีกที)
- **ThaiBulkSMS** — thaibulksms.com
- **SMS Master** — smsmaster.co.th
- **Twilio Verify** — ราคาสูงกว่าแต่ตั้งค่าง่าย รองรับ OTP สำเร็จรูป

**ขั้นตอนทั่วไป**
1. สมัครบัญชี + ยืนยันตัวตนนิติบุคคล (ใช้หนังสือรับรอง หจก. + บัตรประชาชนผู้มีอำนาจ)
2. ขอ **Sender Name** (ชื่อผู้ส่งที่ขึ้นบนมือถือ เช่น `TaladOnline`) — อนุมัติ 1–3 วันทำการ
3. เติมเงิน / ซื้อเครดิต
4. เอา API key มาใส่ Vercel

**บอกผมได้เมื่อสมัครเสร็จ** — ผมต่อ API ให้ ตอนนี้ `src/lib/otp.ts` เตรียมโครงไว้แล้ว

## 🟡 5. ตรวจสลิปอัตโนมัติ (SlipOK) — ยังไม่จำเป็นตอนนี้

จำเป็นเมื่อมีผู้ขายจ่ายค่าสมาชิกเยอะจนตรวจสลิปเองไม่ไหว
ตอนนี้ตรวจมือผ่านหน้า `/admin/payments` ได้อยู่

- **SlipOK** — slipok.com (ราคาต่อครั้ง ไม่มีค่ารายเดือน)
- **EasySlip** — easyslip.com

โค้ดรองรับทั้งสองเจ้าแล้วใน `src/lib/slip.ts` — ได้ API key มาแล้วบอกผม ผมเปิดใช้ให้

## ⚖️ 6. กฎหมาย — ทำคู่ขนานไปได้เลย

> ผมไม่ใช่ทนายและไม่ใช่ที่ปรึกษาภาษี ข้างล่างนี้คือ **รายการที่ต้องไปถามผู้เชี่ยวชาญ**
> ไม่ใช่คำแนะนำทางกฎหมาย กฎเกณฑ์เปลี่ยนได้ ให้ยืนยันกับหน่วยงานอีกครั้ง

### 6.1 จดทะเบียนพาณิชย์อิเล็กทรอนิกส์ (DBD)

- **ใคร:** กรมพัฒนาธุรกิจการค้า (dbd.go.th)
- **เมื่อไหร่:** โดยทั่วไปต้องยื่นภายใน **30 วัน** นับจากเริ่มประกอบธุรกิจออนไลน์
- **ยื่นที่ไหน:** สำนักงานพาณิชย์จังหวัด (เชียงใหม่ ตามที่ตั้ง หจก.) หรือยื่นออนไลน์ผ่านเว็บ DBD
- **เอกสารที่มักต้องใช้:** สำเนาหนังสือรับรองนิติบุคคล, บัตรประชาชนผู้มีอำนาจ, รายละเอียดเว็บไซต์, แผนที่ตั้ง
- **ได้อะไร:** เครื่องหมาย **DBD Registered** เอามาแปะเว็บได้ ช่วยเรื่องความน่าเชื่อถือมาก

### 6.2 ทนายตรวจ ข้อตกลง + นโยบายความเป็นส่วนตัว

ตอนนี้หน้า `/terms` กับ `/privacy` ผมร่างไว้เป็นโครง **ยังไม่ผ่านทนาย**

จุดที่ควรให้ทนายดูเป็นพิเศษ:
- ขอบเขตความรับผิด — เว็บเป็น "ตัวกลางประกาศ" ไม่ใช่คู่สัญญาซื้อขาย
- วิธีรับของแบบ **"โอนก่อน"** — เสี่ยงสุด ถ้าผู้ซื้อโดนโกงบริษัทรับผิดแค่ไหน
- นโยบายคืนเงิน / ระงับบัญชี / จัดการข้อพิพาท
- ข้อความ PDPA ให้ตรงกับข้อมูลที่เก็บจริง (ชื่อ เบอร์ **ที่อยู่จัดส่ง** เลขบัญชี รูปสมุดบัญชี)

### 6.3 PDPA — พ.ร.บ.คุ้มครองข้อมูลส่วนบุคคล

ระบบเก็บข้อมูลอ่อนไหวจริง: ที่อยู่ผู้ซื้อ เบอร์โทร เลขบัญชี รูปหน้าสมุดบัญชี

สิ่งที่ต้องมี:
- ผู้ควบคุมข้อมูล (Data Controller) — ระบุชื่อบริษัทและช่องทางติดต่อในนโยบาย
- ฐานทางกฎหมายในการเก็บแต่ละอย่าง
- ระยะเวลาเก็บและวิธีลบ
- ช่องทางให้เจ้าของข้อมูลขอดู/แก้/ลบข้อมูลตัวเอง ← **ยังไม่มีในระบบ บอกผมได้ ผมทำให้**
- ถ้าเข้าเกณฑ์ อาจต้องแต่งตั้ง **DPO** (เจ้าหน้าที่คุ้มครองข้อมูล) — ให้ทนายประเมิน

### 6.4 ภาษี / VAT

- รายได้เกิน **1.8 ล้านบาท/ปี** โดยทั่วไปต้องจดทะเบียน VAT (ยืนยันกับสรรพากรหรือผู้ทำบัญชี)
- ค่าสมาชิกที่เก็บจากผู้ขายคือ**รายได้ของบริษัท** ต้องออกใบเสร็จ/ใบกำกับภาษี
- ระบบมีหน้าใบเสร็จอยู่แล้วที่ `/sell/receipt/[id]` — ให้ผู้ทำบัญชีดูว่ารูปแบบถูกต้องตามที่สรรพากรกำหนดไหม
- ปรึกษาผู้ทำบัญชีเรื่องภาษีหัก ณ ที่จ่าย ถ้าจะจ่ายค่าคอมมิชชั่น/ค่าจ้างใคร

## ✅ ลำดับที่แนะนำ

```
วันนี้      → ข้อ 1 (Publish LINE)  ← เปิดให้คนนอกใช้ได้ทันที
              ข้อ 2 (Messaging token) ← ระบบแจ้งเตือนเริ่มทำงาน
สัปดาห์นี้  → ข้อ 3 (หมุน service_role) ← ก่อนมีข้อมูลลูกค้าจริง
              ข้อ 6.1 (DBD) + นัดทนาย
ก่อนโปรโมต → ข้อ 4 (SMS OTP)
เมื่อโตขึ้น  → ข้อ 5 (SlipOK), ข้อ 6.4 (VAT)
```

**ข้อ 1 กับ 2 ทำเสร็จเมื่อไหร่ บอกผมได้** — ผมจะทดสอบ LINE Login จากคนนอก
และยิงข้อความแจ้งเตือนจริงให้ดูว่าถึงปลายทางครบทุกจังหวะ
