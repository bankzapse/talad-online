# Phase 2 — Marketplace + Payout (PRD ตั้งต้น)

> โมเดลอนาคต: platform รับเงินจากผู้ซื้อ → หักค่าคอม → จ่ายผู้ขายเป็นรอบ
> แยก path ไว้ที่ `/seller-partner` (ไม่ปนกับ MVP membership ปัจจุบัน)
> ⚠️ ใหญ่กว่า MVP เดิม 3–4 เท่า + มีความเสี่ยงกฎหมาย/การเงินสูง — ทำเป็น Phase แยก

---

## 0. ต้องตัดสินใจก่อนเริ่ม (กำหนด scope/กฎหมาย/ต้นทุนทั้งหมด)
1. **PSP + escrow**: platform **ไม่ควรถือเงินเอง** (เข้าข่าย e-money license ธปท.) → ใช้ **PSP ที่รองรับ marketplace payout / split payment** (PSP ถือเงิน+โอนให้ผู้ขาย)
2. **โมเดลรายได้**: ค่าสมาชิก (เดิม) + **ค่าคอมต่อออเดอร์** (ใหม่) → เก็บทั้งคู่ หรืออย่างใดอย่างหนึ่ง?
3. **KYC ผู้ขาย** ก่อน payout (ยืนยันตัวตน + บัญชี)

## เทียบ PSP (ต้องเช็ครายละเอียด marketplace/split + payout + ค่าธรรมเนียม + ใบอนุญาต)
| PSP | จุดเด่นที่ต้องเช็ค |
|-----|-------------------|
| 2C2P | รองรับ marketplace/split payment, payout |
| Omise (Opn) | Transfers/Recipients API สำหรับจ่ายผู้ขาย |
| GB Prime Pay | ค่าธรรมเนียมถูก, พร้อมเพย์ |
| Beam | split payment |

---

## 1. Flow เงิน
```
ผู้ซื้อสั่ง+จ่าย → PSP ถือเงิน → หักค่าคอม platform
  → บันทึก payable ผู้ขาย → ผู้ซื้อยืนยันรับของ
  → รอบ settlement → PSP โอนผู้ขาย → ออกใบเสร็จ → reconcile
```

## 2. โมดูลที่ต้องสร้าง
- **ผู้ซื้อ**: ตะกร้า, order, จ่ายออนไลน์ (PSP), ติดตามสถานะ, กดรับของ, ขอคืนเงิน
- **ผู้ขาย**: dashboard เงิน (ยอดขาย/ค่าคอม/สุทธิ/available/pending), ประวัติออเดอร์, ตั้งบัญชีรับเงิน, ประวัติ payout, ใบเสร็จ, รายงานยอดขาย
- **แอดมิน**: KYC ยืนยันบัญชี, ตั้งค่าคอม (ระบบ/หมวด/ร้าน), รอบ settlement, payable, สร้าง payout batch, ออกใบเสร็จ, สรุปรายได้, จัดการคืนเงิน/ข้อพิพาท
- **Engine**: **Ledger double-entry** (หัวใจ), Order/Transaction/Commission/Payout models, reconciliation (PSP↔ledger), audit log

## 3. ความเสี่ยง/สิ่งที่ต้องมี
- ใบอนุญาต: ให้ PSP ถือ (platform ไม่ถือเงิน)
- ภาษี: WHT, ใบกำกับภาษี, VAT
- นโยบายคืนเงิน/ข้อพิพาท/ของไม่ตรงปก
- ค่าธรรมเนียม PSP (~2–3%+) เทียบค่าคอมที่เก็บ

## 4. ความปลอดภัยการโอน (Phase 1 ที่เกี่ยวข้อง)
เพราะ Phase 1 ไม่มี escrow → เลี่ยง "โอนแล้วไม่ส่ง" ด้วย:
- คงหลัก **นัดเจอในพื้นที่ + จ่ายสด** + เตือนห้ามโอนล่วงหน้า
- ข้ามจังหวัด → ดัน **COD** (ขนส่งถือเงินแทน = escrow โดยขนส่ง ถูกกฎหมาย)
- escrow จริงมาใน Phase 2 นี้เท่านั้น

## 5. ลำดับแนะนำ
1. ตัดสินใจข้อ 0 (PSP + escrow + โมเดลรายได้)
2. สร้างส่วนไม่แตะเงินก่อน (order/ตะกร้า/dashboard ยอดขาย)
3. เสียบ PSP payout + ledger ทีหลัง
4. KYC + settlement + reconciliation + ใบเสร็จ
