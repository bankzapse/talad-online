# ส่วนที่ 1 — ภาพรวมธุรกิจ

## 1.1 โมเดล
- กระดานประกาศซื้อขาย **ของสด + สินค้ามือสอง + อื่นๆ** ที่มีในตลาด
- ผู้ซื้อ–ผู้ขาย **นัดเจอ จ่ายเงินสดกันเอง** → ระบบไม่แตะเงินซื้อขาย = ความเสี่ยงต่ำ ไม่ติดใบอนุญาต e-money
- **รายได้:** ค่าสมาชิกผู้ขาย — ทดลองฟรี 30 วัน แล้วเลือกซื้อแพ็ก

## 1.2 แพ็กสมาชิก (แก้ราคาได้จาก admin)
| แพ็ก | ราคา | ตกเดือนละ | ประหยัด |
|------|------|-----------|---------|
| 7 วัน | 39 | ~167 | — |
| รายเดือน | 99 | 99 | ฐาน |
| 3 เดือน | 279 | 93 | ~6% |
| 6 เดือน | 539 | ~90 | ~9% |
| รายปี | 999 | ~83 | ~16% |

## 1.3 จุดต่างจากคู่แข่ง (ต้องตอบให้ได้ก่อนขาย)
ทำไมแม่ค้าจ่าย 99฿/เดือน แทน Facebook/Kaidee ที่ฟรี:
- เจาะ **ตลาด/พื้นที่เฉพาะ** ที่แม่ค้ากลุ่มเดียวกันอยู่จริง
- **filter พื้นที่ + เรียงราคา/หน่วย** ที่กลุ่ม FB ทำไม่ได้
- **แจ้งเตือนเข้า LINE ทันที** เมื่อมีคนสนใจ
> ⚠️ ตอบไม่ชัด = ความเสี่ยงเชิงธุรกิจ ต้องเคลียร์ก่อนเก็บเงิน

# ส่วนที่ 2 — ฟีเจอร์ MVP

## 🏪 ฝั่งผู้ขาย
- ล็อกอิน **LINE Login** + บังคับเพิ่มเพื่อน LINE OA
- **OTP ยืนยันเบอร์ตอนสมัคร** → 1 เบอร์ = 1 สิทธิ์ทดลอง (กันสมัครซ้ำ)
- ลงสินค้าผ่าน **LIFF form**: รูป, ราคา + **หน่วย**, หมวด, **พื้นที่/ตลาด (dropdown)**
- จัดการประกาศ: แก้ / ปิด / ต่ออายุ
- แจ้งเตือนทาง LINE เมื่อมีผู้ซื้อสนใจ
- **บล็อก/รายงานผู้ซื้อกวน** ได้
- สมาชิก: ทดลอง 30 วัน → เลือกแพ็ก → **PromptPay QR + อัปสลิป** → เตือนก่อนหมด

## 🛒 ฝั่งผู้ซื้อ
- **filter หมวด+พื้นที่** / **เรียงราคา** (แสดงหน่วยชัด เช่น "45 ฿/กก.")
- ค้นหาแบบง่าย (ยังไม่ full-text ไทยใน MVP)
- ปุ่ม **"ติดต่อ"** → push เข้า LINE ผู้ขาย
  - **ต้องล็อกอินก่อนกด + rate-limit** (กันปั่น push + คุมต้นทุน)
  - **Fallback:** โควตา push หมด → แสดงช่องทางติดต่อให้กดเอง
- ดูได้โดยไม่ต้องสมัคร (สมัครเมื่อจะติดต่อ)

## 🧾 สัญญาณความน่าเชื่อถือ (MVP — ยังไม่ต้องมีรีวิวเต็ม)
- badge **"ยืนยันเบอร์แล้ว"** + **วันเข้าร่วม** + **จำนวนประกาศ**

## 🛠️ ฝั่ง Admin
- อนุมัติ/ลบประกาศ, แบน/ปลดแบน
- **คิวตรวจประกาศที่ถูก report / ติด blocklist**
- จัดการหมวด + **พื้นที่/ตลาด (dropdown)**
- **แก้ราคา/เพิ่ม/ปิดแพ็ก** จากหน้าจอ
- **ยืนยันการจ่าย** (ดูผลตรวจสลิป + กดยืนยันเคส edge)
- **ปรับวันหมดอายุ + log** (grace/คืนเงิน/ระบบพลาด)
- ดูรายได้ + **analytics** (ประกาศใหม่/ยอดติดต่อ/conversion)

## กติกาสำคัญ
- หมดอายุ = **ซ่อน ไม่ลบ** (จ่ายแล้วกลับมาทันที)
- ต่ออายุก่อนหมด = **ต่อท้ายวันเดิม** (ระวัง edge case timezone/สิ้นเดือน)
- ขึ้นราคา = **มีผลรอบใหม่เท่านั้น**
- เรียงราคา = แสดงหน่วยชัด กันเทียบข้ามหน่วย

## Tech stack
**Next.js + Supabase** (auth/DB/รูป) + **LINE** (Login/Messaging/LIFF) + **PromptPay QR + slip verification**

# ส่วนที่ 4 — ตัดสินใจ 3 Blocker (ประหยัด + เสี่ยงน้อยสุด)

## Blocker #1 — ยืนยันเงิน PromptPay
| ทางเลือก | ต้นทุน | เสี่ยง | สรุป |
|----------|--------|--------|------|
| Payment gateway | หัก 2–3%+/รายการ | ต่ำ แต่แพงเมื่อยอดเล็ก | เกินจำเป็น |
| ตรวจสลิปมือล้วน | ฿0 | **สูง** (สลิปปลอม+ช้า) | ไม่แนะนำเดี่ยว |
| **✅ อัปสลิป + API ตรวจสลิปไทย** | **ถูก/ไม่หัก %** | **ต่ำ** (เช็คธนาคารจริง) | **แนะนำ** |

→ ผู้ขายโอน → อัปสลิป → API ยืนยันกับธนาคารอัตโนมัติ → เปิดสิทธิ์ทันที, เคส edge → admin กดมือ

## Blocker #2 — Cost model (LINE push + OTP)
- ต้นทุนแปรผันจริง = push ปุ่มติดต่อ → **คุมด้วย gate ปุ่ม** (ล็อกอิน+rate-limit) ยิงตรง 2 ปัญหา (ต้นทุน+spam)
- OTP = เสียครั้งเดียวตอนสมัคร (bounded)
- เริ่ม LINE free tier ตลาดนำร่อง → upgrade เมื่อจำเป็น + fallback เมื่อโควตาหมด
- **สูตร:** ผู้ขาย 1 คน (99฿) รับ push ได้ ≈ 99 ÷ ค่า push ก่อนขาดทุน → คุมจำนวน/คน/เดือนให้ต่ำกว่านั้น

## Blocker #3 — เอกสารกฎหมาย (อันบังคับ+ถูกก่อน)
| งาน | จำเป็น | ทำให้ประหยัด |
|-----|--------|--------------|
| จดทะเบียนพาณิชย์ฯ (DBD) | บังคับ | ทำก่อน ถูก เลี่ยงค่าปรับ |
| PDPA + Terms | บังคับ | template + ทนายรีวิว |
| Disclaimer อาหาร + ของต้องห้าม | จำเป็น | เขียนอิงเทมเพลต + รีวิวรวม |
| ปรึกษาบัญชี VAT/ภาษี | จำเป็น | ครั้งเดียว วางโครงถูกแต่แรก |

# ส่วนที่ 6 — ต้นทุนจริง + Break-even

> ⚠️ ประมาณการต้นปี 2026 — เช็คหน้าราคาปัจจุบันก่อนตัดสินใจจริง

## 6.1 ราคาต่อหน่วย (ประมาณการ)
| รายการ | ราคา | หมายเหตุ |
|--------|------|----------|
| Vercel | Hobby ฟรี / **Pro ~฿700/ด.** | Pro สำหรับ prod |
| Supabase | Free / **Pro ~฿875/ด.** | Pro ได้ backup |
| LINE OA Light | **~฿230/ด.** (~15k ข้อความ) | — |
| LINE OA Standard | **~฿1,200/ด.** (~35k) | เกิน ~฿0.06/ข้อความ |
| SMS OTP | **~฿0.35–0.50/ครั้ง** | ครั้งเดียวตอนสมัคร |
| ตรวจสลิป | **~฿0.5–1/ครั้ง** หรือแพ็ก ~฿250/ด. | ไม่หัก % |
| Domain | **~฿40/ด.** | — |

> 💡 LINE push ส่วนเกินถูกมาก (~฿0.06) → **ต้นทุนหลักคือ fixed cost ไม่ใช่ push** (ถ้า gate ปุ่มไว้)

## 6.2 สมมติฐาน
ค่าสมาชิกเฉลี่ย ฿99/เดือน · conversion ทดลอง→จ่าย 25% · push ~20–30 ครั้ง/ผู้ขาย/เดือน (หลัง gate)

*ก่อนหักค่าคน/การตลาด/ภาษี

## 6.4 จุดคุ้มทุน
- **คุ้มค่าระบบอย่างเดียว** → ~22 ผู้ขายจ่ายเงิน (ผู้ขายลอง ~88)
- **คุ้มเมื่อรวมคนดูแล 1 คน (~฿8,000)** → **~100 ผู้ขายจ่ายเงิน** ← เป้าเลี้ยงตัวเองได้
- รวมการตลาด (~฿10,000) → ~203 ผู้ขายจ่ายเงิน

## 6.5 สิ่งที่ตัวเลขบอก
1. ต้นทุนแปรผัน (push/OTP/สลิป) **น้อยมาก** → ความกลัว "push ขาดทุน" ไม่จริงถ้า gate ปุ่มไว้
2. ตัวขับต้นทุนจริง = **fixed cost + ค่าคน**
3. เริ่มด้วย free tier ช่วง dev → ขยับ Pro ตอนเปิดจริง
4. **🎯 เป้าตลาดนำร่อง: ผู้ขายจ่ายเงิน ~100 คน = ธุรกิจเลี้ยงตัวเองได้** แล้วค่อยขยายตลาดที่ 2

## 6.6 ต้นทุนก้อนเดียว (one-time)
DBD (หลักร้อย) · ทนายรีวิว PDPA/Terms (หลักพัน–หมื่นต้น) · ปรึกษาบัญชี (ครั้งเดียว) · ค่า dev Phase 1 (แล้วแต่ทีม)

# ส่วนที่ 8 — ลำดับการปล่อย (Phase)

**เริ่มตลาดเดียว** → ชวนลองฟรี → เก็บ feedback → แก้ให้นิ่ง → ขยาย

- **Phase 1 (MVP):** LINE Login + OTP + LIFF ลงสินค้า + เว็บ filter/เรียงราคา + ปุ่มติดต่อ (gate+rate-limit+fallback) + trust badge + report/blocklist + สมาชิก + จ่าย QR+ตรวจสลิป + admin (คิว/พื้นที่/ปรับวันหมดอายุ/analytics) + RLS + idempotency
- **Phase 2:** ค่าดันประกาศ + รีวิวผู้ขาย + ของสดหมดอายุอัตโนมัติ + full-text ไทย + AI image moderation
- **Phase 3:** แจ้งเตือนเพิ่มเติม + แอปมือถือ

# ส่วนที่ 9 — สรุปสุดท้าย

- **ฟีเจอร์ + logic + ช่องโหว่:** ปิดครบระดับเริ่ม dev ได้ ✅
- **3 blocker มีคำตอบแล้ว:** เงิน → อัปสลิป+API ตรวจสลิป · Cost → gate ปุ่ม+free tier+OTP ครั้งเดียว · กฎหมาย → DBD ก่อน+template/ทนาย+บัญชีครั้งเดียว
- **ต้นทุน:** แปรผันน้อยมาก, ตัวหลักคือ fixed+ค่าคน → **คุ้มที่ ~100 ผู้ขายจ่ายเงิน**
- **ของต้องห้าม:** จัด 6 กลุ่ม + blocklist พร้อมใช้
- **เหลือการบ้านเชิงธุรกิจ** (ไม่ใช่โค้ด): เคลียร์ value prop + จังหวะ monetization ในตลาดนำร่อง

> 🎯 เป้าเดียวที่ควรโฟกัส: **ตลาดนำร่อง 1 แห่ง → ผู้ขายจ่ายเงินให้ถึง ~100 คน** = พิสูจน์ว่าโมเดลเวิร์ก + ธุรกิจเลี้ยงตัวเองได้ ก่อนขยาย
