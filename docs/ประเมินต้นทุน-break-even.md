> ⚠️ ตัวเลขเป็น **ประมาณการ ณ ต้นปี 2026** — ราคาผู้ให้บริการเปลี่ยนได้ ต้องเช็คหน้าราคาปัจจุบันก่อนตัดสินใจจริง

## 2. สมมติฐานพฤติกรรม

- ค่าสมาชิกเฉลี่ย (blended) ใช้ **฿99/เดือน** (คิดแบบอนุรักษ์นิยม; แพ็กยาวจะได้มากกว่านี้)
- Conversion ทดลอง→จ่าย ≈ **25%**
- ผู้ขาย active (จ่าย+ทดลอง) รับ push จากปุ่มติดต่อ ≈ **20–30 ครั้ง/คน/เดือน** (หลัง gate + rate-limit)
- OTP = จำนวนผู้ขายใหม่/เดือน
- ตรวจสลิป = จำนวนการจ่าย/เดือน

## 4. จุดคุ้มทุน (Break-even)

### ระดับ 1 — คุ้ม "ค่าระบบ" อย่างเดียว
- Fixed infra ≈ **฿2,100/เดือน**
- Break-even = 2,100 ÷ 99 ≈ **22 ผู้ขายที่จ่ายเงิน**
- ที่ conversion 25% → ต้องมีผู้ขายลอง ~88 คน

### ระดับ 2 — คุ้มเมื่อรวม "คนดูแล 1 คน (part-time)"
- + ค่าคน ~฿8,000/เดือน → Fixed รวม ≈ ฿10,100
- Break-even = 10,100 ÷ 99 ≈ **~102 ผู้ขายที่จ่ายเงิน**
- ที่ conversion 25% → ต้องมีผู้ขายลอง ~408 คน

### ระดับ 3 — รวมการตลาดด้วย (เช่น +฿10,000/เดือน)
- Fixed รวม ≈ ฿20,100 → Break-even ≈ **~203 ผู้ขายที่จ่ายเงิน**

## 5. ข้อสรุป / สิ่งที่ตัวเลขบอก

1. **ต้นทุนแปรผัน (push/OTP/สลิป) น้อยมาก** — รวมกันหลักร้อยบาท/เดือน แม้ที่ scale
   → ความกลัว "push ทำขาดทุน" **ไม่จริง** ถ้า gate ปุ่มติดต่อไว้ (ทำแล้วใน v3)
2. **ตัวขับต้นทุนจริง = Fixed cost** → hosting/plan (~฿2,100) + **ค่าคน** (ตัวใหญ่สุดเมื่อโต)
3. **คุ้มค่าระบบที่ ~22 ผู้ขายจ่ายเงิน** — เป้าที่ตลาดนำร่องเดียวทำได้จริง
4. **คุ้มแบบมีคนดูแลที่ ~100 ผู้ขายจ่ายเงิน** — นี่คือเป้าที่ทำให้ธุรกิจ "เลี้ยงตัวเองได้"
5. เริ่มด้วย **free tier (Vercel Hobby / Supabase Free / LINE Free)** ช่วง dev+ทดลอง แล้วค่อยขยับเป็น Pro ตอนเปิดจริง → ประหยัดช่วงแรก

### 🎯 เป้าที่ควรตั้ง
> ตลาดนำร่อง: หา **ผู้ขายจ่ายเงินให้ถึง ~100 คน** = ธุรกิจเลี้ยงตัวเอง (รวมค่าคน 1 คน) แล้วค่อยขยายตลาดที่ 2
